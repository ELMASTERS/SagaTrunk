Type extensions:

By default saga comes with a default fileformats xml or csv. But often it happens you have you're own fileformat or want to add encryption. Well don't worry because sags supports type extension from withing the scripting you're allowed to write custom override types and use those a fundamental base for your server.

You can speciafy a type overloading by using the type parameter in your configuration files.

Example: 
<Saga.Factory.Addition type="Plugins.ExtendedAddition">
	<Files>
		<add path="../Data/addition.csv" />
	</Files>
</Saga.Factory.Addition>

Saga.Factory.Addition.

Summary

The addition factory loads in a massive amount of data defining what type of side-affects certain items and skill have on your players status. Most of the buffs/debuffs and items use this as a fundamental base. The addition factory by default has not other options besides sharded file loading.

Configuration: 

<configuration>
	<configSections>
		<section name="Saga.Factory.Addition" type="Configuration.Factory_Addition, Saga.Map"/>		
	</configSections>
	
	<!-- Some other configuration options here -->
	
	<Saga.Factory.Addition>
		<Files>
			<add path="../Data/addition.csv" />
		</Files>
	</Saga.Factory.Addition>
</configuration>

Saga.Factory.Exptable.

Summary

The expierence table loads in the data which defines how many expierence is required for each level. Note that the expierence table works in conjunction with the client, so if the expierence table is wrong for some reason it may result in bad gameplay.

Addition parameters

cexp You can use cexp to set the rate modifier. This is equavelent to exp = exp_reward * rate;
jexp You can use jexp to set the rate modifier. This is equavelent to exp = exp_reward * rate;
wexp You can use wexp to set the rate modifier. This is equavelent to exp = exp_reward * rate;

Configuration: 

<configuration>
	<configSections>
		<section name="Saga.Factory.Exptable" type="Configuration.Factory_Exptable, Saga.Map"/>		
	</configSections>
	
	<!-- Some other configuration options here -->
	
	<Saga.Factory.Exptable [cexp="integer"] [jexp="integer"] [wexp="integer"]>
		<Files>
			<add path="../Data/expierence.csv" />
		</Files>
	</Saga.Factory.Exptable>
</configuration>

Saga.Factory.Itemdrops.

Summary

The item drop manager is not a crucial component. It has been added to make it easier for test-servers to quickly modify item drops, and reload them on demand. Could also server as part from a general database extraction so we can keep things as current as posible

Configuration: 

<configuration>
	<configSections>
		<section name="Saga.Factory.Itemdrops" type="Configuration.Factory_Itemdrops, Saga.Map"/>		
	</configSections>
	
	<!-- Some other configuration options here -->
	
	<Saga.Factory.Itemdrops>
		<Files>
			<add path="../Data/item_drops.csv" />
		</Files>
	</Saga.Factory.Itemdrops>
</configuration>

Saga.Factory.Items

Summary

The item factory is a very important class. It contains all item information: categories for the auction, sell prices, if a item is tradeable. By which classes it can be used.

Configuration: 

<configuration>
	<configSections>
		<section name="Saga.Factory.Items" type="Configuration.Factory_Items, Saga.Map"/>		
	</configSections>
	
	<!-- Some other configuration options here -->
	
	<Saga.Factory.Items>
		<Files>
			<add path="../Data/item_data.xml" reader="xml"  />
		</Files>
	</Saga.Factory.Items>
</configuration>
