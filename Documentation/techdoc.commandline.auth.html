account -create

Remarks:

This will create an account to the database. The account will be a regulair account with and no gm level.

Example:

The following code sample illustrates the use of account -create:

Server instance
account -create phr34k918 ha2jumi female
account created

account -ip

Remarks:

Displays the last IP address that was used by the player.

Example:

The following code sample illustrates the use of account -ip:

Server instance
account -ip phr34k
User last ip adress: 127.0.0.1
account -ip phr34k918
User last ip adress: 0.0.0.0

account -isonline

Remarks:

Checks if the specified account is online right now.

Example:

The following code sample illustrates the use of account -isonline:

Server instance
account -isonline phr34k918
User is: offline

account -kick

Remarks:

This will look up your active session id. If this field is set to non-zero. It will contact the last played world server to kick your session.

Example:

The following code sample illustrates the use of account -kick:

Server instance
account -kick phr34k

account -time

Remarks:

Computes the actual ammount the user has played. If the user currently is logged in the added gametime is returned is not updated.

Example:

The following code sample illustrates the use of account -time:

Server instance
account -time phr34k
User has played: 0 days - 0 hours - 6 minutes
account -time phr34k918
User has never played

acl -add

Remarks:

This will add a rule policy to the access control list.

Example:

The following code sample illustrates the use of acl -add:

Server instance
acl -add allow 0.0.0.0/0
Entry added

acl -dump

Remarks:

This will dump a list of banned ip and filtermasks. Optionally you can use an ip address as third argument to check for masks that affect the speciafied ip address. It creates a file called acl.txt in the same directory as the binairy.

Example:

The following code sample illustrates the use of acl -dump:

Server instance
acl -dump 127.0.0.1

Output generated in: acl.txt

Access Control Lists
===================================================================================

Rule policy:
A explicit allow declaration can override a deny operation
All ip's are implicit allowed.

Filtering all entries on matching: 127.0.0.1


RULE#	ALLOW/DENY          	IP             	MASK           
1	DENY                	0.0.0.0        	0.0.0.0        
2	ALLOW               	127.0.0.1      	255.255.255.255

acl -remove

Remarks:

This will remove a previously added rule policy to the access control list.

Example:

The following code sample illustrates the use of acl -remove:

Server instance
acl -remove allow 0.0.0.0/0
Entry removed
